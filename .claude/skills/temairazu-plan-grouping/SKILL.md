# 手間いらず プラングループ・デライブドグループ 設定Skill

## このSkillの目的

施設がバラバラに値付けしているプランのExcel/CSVデータを読み取り、以下を出力する：

1. **プラングループ設計案** — 同じ料金軸のプランをまとめてグループ化する
2. **デライブドグループ設計案** — 素泊まりをBAR（基準）として、他プランとの差額を算出する
3. **手間いらずへの設定手順書** — 実際の操作手順をステップごとに出力する

---

## Step 1: ファイルの読み取りと整理

### ⚠️ ファイル形式の注意点

手間いらずからダウンロードされる料金一覧ファイルは **拡張子が `.xls` だが中身はHTMLテーブル** になっている。
通常のExcel読み取り（xlrd等）では `XLRDError` が発生するため、必ずHTMLとして読み取ること。

### ファイルの読み取り手順（Pythonコード）

```python
from bs4 import BeautifulSoup
from collections import defaultdict

with open('pricelist.xls', 'r', encoding='utf-8') as f:
    html = f.read()

soup = BeautifulSoup(html, 'html.parser')
rows = soup.find_all('tr')

# データ行は4行目（index=3）以降。空行（部屋料金ヘッダー行）はスキップ
data = []
for row in rows[3:]:
    cells = [td.get_text(strip=True) for td in row.find_all('td')]
    if len(cells) >= 5 and cells[0] and cells[1] and cells[2] and cells[3]:
        data.append({
            'site':   cells[0],   # 予約サイト名
            'room':   cells[1],   # 部屋タイプ名（サイト独自）
            'plan':   cells[2],   # プラン名（ID: プラン名 形式）
            'rank':   cells[3],   # 料金ランク（A〜Q）
            'price':  cells[4],   # 提供料金（部屋料金）
        })
```

### カラム構造

| 列 | 内容 | 補足 |
|---|---|---|
| 予約サイト | OTA名 | 楽天トラベル、じゃらん、Booking.com など14サイト |
| 部屋名 | **各サイト独自の部屋タイプ名** | サイトをまたいで同一部屋かどうかは別途対応表が必要 |
| プラン名 | `ID: プラン名` の形式 | 例：`3150176: お気軽に♪【素泊まり】` |
| 料金ランク | A〜Qの17段階 | **分析にはランクAを基準として使う** |
| 提供料金 | 販売価格（部屋料金） | 0円はそのランクで販売停止の意味 |

### 料金ランクについて

- ランクA = その部屋・プランの最高価格設定
- ランクが下がるにつれて価格も下がる（Q が最低）
- **プラングループ設計の比較・差額計算は必ずランクAで行う**

### 部屋タイプの名寄せが必要

各OTAで同じ部屋でも名前が異なる。分析を始める前に、ユーザーに以下を確認する：

> 「手間いらず側の部屋マスタ名と、各OTAの部屋タイプ名の対応表をいただけますか？」

対応表の例：

| 手間いらず部屋マスタ | 楽天 | じゃらん | Booking.com | Expedia |
|---|---|---|---|---|
| シングル禁煙 | s1: スタンダードシングル【禁煙】 | 0259273: スタンダードシングル【禁煙】 | Single Room Non-Smoking | Standard Single Room |

対応表が得られたら、データを手間いらず部屋マスタ名で統合して分析する。

## Step 2: プラングループ設計

### グルーピングの基本方針

プランを「食事内容」で分類するのが基本。よくある分類：

- **素泊まり** → BARグループ（後述）
- **朝食付き** → デライブドグループ
- **夕食付き** → デライブドグループ
- **朝夕食付き** → デライブドグループ
- **特典付き（アメニティ等）** → デライブドグループまたは別グループ

### 同一グループにまとめる条件

- 複数OTAにまたがる同じ食事条件のプランは同一グループにまとめられる
- 部屋タイプが異なれば別グループ（例：シングルの素泊まり vs ダブルの素泊まり）
- MC（人数売り）とRC（部屋売り）が混在する場合は注意が必要（後述）

### MC/RC混在の注意点

同一プラングループにMC（人数売り）とRC（部屋売り）が混在した場合、
手間いらずは以下の計算を自動で行う：

> MCプランへの反映料金 = プラングループの料金 ÷ 人数
> 例）RC2の料金10,000円 → MCには5,000円が反映

これが意図しない料金になる場合は、MとRでグループを分けることを提案する。

### 出力フォーマット：プラングループ設計案

```
【プラングループ設計案】

■ グループ1：素泊まり（BARグループ）
  対象プラン：
  - [OTA名] [部屋タイプ名] [プラン名]
  - [OTA名] [部屋タイプ名] [プラン名]
  料金入力方法：1名あたり / 1室あたり（どちらか）

■ グループ2：朝食付き（デライブドグループ）
  対象プラン：
  - [OTA名] [部屋タイプ名] [プラン名]
  BARグループとの差額：+X,XXX円

（以下同様）
```

## Step 3: デライブドグループ設計

### BARグループの決め方

**素泊まりプランのグループがBAR（Best Available Rate）になる。**

BARグループ = 手間いらずの料金カレンダーで直接料金を入力・管理するグループ

### デライブドグループの差額算出

各プラングループの料金からBARグループの料金を引いて差額を算出する。

```
差額 = 各プランの基準料金 - 素泊まりの基準料金
```

例：
- 素泊まり：8,000円（BAR）
- 朝食付き：10,000円 → 差額 +2,000円
- 朝夕食付き：15,000円 → 差額 +7,000円

### 差額の設定方法

手間いらずのデライブドプライス設定では以下から選択：

| 種類 | 説明 | 使いどころ |
|------|------|-----------|
| M（1名あたり差額） | 人数×差額が加算 | 食事付きプラン（1名ずつ料金が変わる場合） |
| R（1室あたり差額） | 固定差額を加算 | 固定オプション（Wi-Fiなど部屋単位） |
| ％（割増・割引） | BAR料金に対してパーセント | 会員割引など |

**食事付きプランは基本的に「M（1名あたり）」を推奨。**

### オプション機能の活用

差額が複数要素で構成される場合（例：朝食+駐車場）はオプションを組み合わせて設定できる。
オプション料金の計算順：差額 → オプション加算 → 最終調整（%）

### 出力フォーマット：デライブドグループ設計案

```
【デライブドグループ設計案】

BARグループ：素泊まり（グループ1）

■ デライブドグループ：朝食付き（グループ2）
  差額設定：M（1名あたり） +X,XXX円
  ※素泊まり料金にこの差額が自動加算される

■ デライブドグループ：朝夕食付き（グループ3）
  差額設定：M（1名あたり） +X,XXX円

（以下同様）
```

## Step 4: 手間いらず設定手順書の出力

### 前提条件の確認チェックリスト

以下が完了済みか必ず確認する：

- [ ] 料金カレンダー設定済み
- [ ] 各OTAの部屋・プラン情報を手間いらずで取得済み
- [ ] 手間いらず側の部屋マスタへの紐付け済み

### 手順書出力フォーマット

```
【手間いらず設定手順書】

■ STEP 1：プラングループ利用設定
  設定＞基本設定＞プラングループ利用設定
  1. 「プラングループを使用する」を選択
  2. 部屋詳細設定の適用有無を設定
     ※通常は「無効」のまま。個別に部屋詳細設定を引き継ぎたい場合のみ有効
  3. 料金調整方法を選択（端数処理方法）
  4. 税サ設定を入力 → リフレッシュ更新を実行
  5. デライブドプライシング：「利用する」を選択

■ STEP 2：プラングループの作成
  料金コントロール＞プラングループ一覧＞追加

  [グループ1：素泊まり（BAR）]
  - 部屋マスタ：[部屋名]
  - 料金入力方法：[1名あたり/1室あたり]
  - プラングループ名：素泊まり
  → 登録後、対象プランを紐付け（「全て」クリックで複数サイト一括紐付け可）

  [グループ2：朝食付き（デライブド）]
  - 部屋マスタ：[部屋名]
  - 料金入力方法：[1名あたり/1室あたり]
  - プラングループ名：朝食付き
  → 登録後、対象プランを紐付け

  ※ 登録後に部屋マスタ・料金入力方法の変更は不可

■ STEP 3：デライブドグループ設定
  料金コントロール＞プラングループ一覧＞デライブドグループ設定

  1. BARグループ：「素泊まり」を選択
  2. デライブドプライス設定をクリック
  3. 朝食付きグループに対して差額設定：
     M（1名あたり） +[X,XXX]円 を入力
  4. 「再計算」で参考料金を確認
  5. 「登録」をクリック
  ※ 再計算だけでは登録されないので注意

■ STEP 4：料金の入力と反映
  料金コントロール＞料金カレンダー（BARグループ）
  - 素泊まりグループのカレンダーに料金を入力
  - 保存後、デライブドグループは自動計算で反映される
  - 反映料金の確認：ツール＞料金一覧
```

## 注意事項・よくある落とし穴

### デライブドグループの注意点

- デライブドグループには**ターゲットプライス設定は反映されない**
- 部屋プラン一覧上の料金表示が「料金未設定」や古い料金になることがある（表示バグ）。予約サイト上には正しく反映される
- BARグループ設定を解除した後は**プラングループ販売料金再計算**を必ず実行する

### プラングループ移行ツールの活用

すでに「料金グループ」で管理していた施設の場合、移行ツールが使える：
- 設定＞基本設定＞プラングループ利用設定の画面最下部に「プラングループ移行ツール」が表示される
- 既存の料金グループをベースにプラングループを自動作成できる

### 税サ設定の注意

税サ設定を変更した後は**必ずリフレッシュ更新**を実行すること。
設定した税サ金額はサイトへの料金送信時に適用される。

## このSkillを使うときのフロー

1. ユーザーから `.xls` ファイルを受け取る
2. **Step 1**：HTMLとして読み取り → OTA・部屋・プラン・料金ランクAを抽出
3. 部屋マスタ対応表をユーザーに確認（未提供の場合）
4. **Step 2**：対応表を使ってデータを統合 → プラングループ設計案を提示・確認
5. **Step 3**：素泊まりをBARとしてデライブド差額を計算 → 設計案を提示・確認
6. **Step 4**：手間いらずへの設定手順書を出力
7. ユーザーのフィードバックに基づいて修正
